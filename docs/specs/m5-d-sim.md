# Tetorial 명세: 하이라이트 토글 (@tetorial/sim)

> 상태: M5 W5-2a (#54 U-7의 sim 축) · 관련 모듈: **packages/sim 단독**
> 역할: **하이라이트 도구를 켜기 전용에서 켜기/끄기 토글로 바꾼다 (스트로크 단위 모드 확정).**
>
> 현행 (2026-07-18 실측): `OverlayBuffer.set(x, y, on)`(`src/overlays.ts`)은 켜기/끄기를 다 받지만, `strokeTo`(`src/authoring.ts` — `s.tool.kind === "highlight"` 분기)가 항상 `true`로만 호출한다 — 끄기 경로 미구현(버그 아님, D-9 예약 상태였음 → D-21로 구현 확정).

## 1. 경계 / 책임

- 수정 범위는 **packages/sim뿐이다.** `authoring.ts`의 스트로크 처리와 `Tool` 타입만.
- 하지 않는 것: 렌더링(renderer, W5-2a 병렬 세션) · 팔레트/우클릭 UI(apps/web, W5-2b) · 스키마 변경(overlays 인코딩 불변 — `@tetorial/types` 접근 금지).
- 결정론 유지: `Math.random`·`Date` 금지(기존 규칙).

## 2. API — Tool 확장

```ts
/** 그리기 도구 — cell/erase는 보드, highlight는 오버레이 */
export type Tool =
  | { kind: "cell"; v: Cell }
  | { kind: "erase" }
  | { kind: "highlight"; force?: "on" | "off" };
```

- `force`는 W5-2b(apps/web)의 우클릭 지우기가 쓸 예약 모드다: `"off"` = 이 스트로크는 끄기만, `"on"` = 켜기만.
- `force` 부재(기본) = **토글 모드**.

## 3. 토글 규범(規範)

- **스트로크 단위 모드 확정**: `force` 부재 시, 스트로크에서 **처음 만나는 유효 셀**(OverlayBuffer 범위 안)의 현재 상태를 읽어 그 **반전**을 이 스트로크의 모드로 확정하고, 스트로크가 끝날 때까지 모든 셀에 같은 모드를 적용한다.
  - 셀별 개별 토글 금지 — 드래그가 같은 영역을 지그재그로 오가도 깜빡이지 않는다. (중복 셀은 기존 `seen` 규약이 이미 무시하지만, 모드 자체가 셀마다 뒤집히면 안 된다는 뜻이다.)
  - 단일 클릭(셀 1개 스트로크)은 자연히 "같은 셀에서 켜기/끄기 토글"이 된다.
- 첫 셀이 범위 밖이면 모드 미확정 상태를 유지하고, 다음 유효 셀에서 확정한다.
- 변경 없음(`set`이 false 반환 — 이미 그 상태)이면 `changed`를 세우지 않는다 — 언두 스택에 빈 스트로크가 쌓이지 않는 기존 규약 유지.
- 언두: 스트로크 1회 = 언두 1단위 (기존 규약 불변).

## 수용 기준 (작업 세션 완료 조건)

- **S-10 하이라이트 토글**: 하이라이트 도구가 같은 셀에서 켜기/끄기 토글로 동작한다 — 검증: ① 같은 셀 2회 스트로크 = 켜짐→꺼짐 ② 켜진 셀에서 시작한 드래그는 경로 전체를 끄기(꺼진 셀 포함 지그재그에서 모드 불변) ③ `force:"off"`/`"on"` 강제 모드 ④ 무변경 스트로크는 언두 단위를 만들지 않음 ⑤ 토글 스트로크의 undo/redo 왕복
