# Tetorial 명세: 1vs1 양보드 동시 재생 (M6-B, @tetorial/web)

> 상태: 작업 중 · 관련 모듈: apps/web (단독)
> 역할: **1vs1 리플레이에서 두 플레이어의 보드를 동시에 재생한다. 단일 보드 + 플레이어 셀렉터 구조를 대체한다.**
> 근거: #15 (게이트 12항의 기원 — 단일 보드+셀렉터는 요청된 적 없는 구현이었다). 확장안(스왑·왼쪽 진입)은 상담 정리 2026-07-18 소유자 채택. 전제인 공통 HUD(#53)·비모달 전환(#55, M6-A)은 완료.

## 1. 경계 / 책임

- 수정 범위는 **apps/web뿐**이다. `packages/*` 수정 금지 — `@tetorial/replay-tetrio` 변경이 필요하다고 판단되면 **수정하지 말고 QUESTIONS.md에 근거를 적고 해당 부분을 보류**하라. 총괄이 명세를 개정한다.
- 스키마·딥링크·저장 와이어 포맷 무관 웨이브다. `Origin`은 이미 `player`를 갖고 있다(#15 — 동결 창 밖).
- 시뮬레이터·노트 저작·업로드 기능은 무변경. 진입 지점의 보드 선택 규칙만 이 명세가 정한다(§4).

## 2. 현행 구조 (출발점)

- `ReplayIsland`가 `(round, player)` 하나를 골라 `createPlaybackSession`(lib/playback-session.ts)으로 세션을 만들고, `RoundPlayerSelect`가 라운드·**플레이어** 셀렉터를 제공한다.
- `PlaybackSession` = `createPlayback(doc, {round, player})` 컨트롤러 + `PlaybackClock`(주입식 타이머) 셸. 시계 하나가 컨트롤러 하나를 구동한다.
- 보드·HUD는 `GameHud`+`BoardCanvas` 1조. 분기(`BranchBar`)는 현재 세션의 `(round, player, frame)`으로 진입한다.

## 3. 목표 동작 — 양보드 재생

- **1vs1 라운드(플레이어 2명)**: 두 플레이어의 보드+`GameHud`를 좌우로 동시 표시·동시 재생한다. `RoundPlayerSelect`의 **플레이어 셀렉터는 1vs1에서 제거**된다(라운드 선택은 유지).
- **솔로(ttr, 플레이어 1명)**: 현행 단일 보드 유지. 변경 없음.
- 규범(規範) — **프레임 동기(AW-38)**: 재생·일시정지·seek·배속·프레임 스텝은 항상 두 보드에 함께 적용된다. **정지 상태(일시정지·seek 직후·스텝 직후)에서 두 컨트롤러의 `frame`은 언제나 동일**해야 한다. 재생 중 어긋남이 구조적으로 불가능하도록 **구동원(시계)은 하나**로 하라 — 시계 2개 병행은 프레임 드리프트를 만든다. 구체 방식(마스터 세션 + 미러 등)은 구현 재량.
- 규범(規範) — **총프레임 상이**: 1vs1은 한쪽이 먼저 끝날 수 있다(`totalFrames` 상이). 슬라이더 범위는 **두 보드의 max(totalFrames)**, 짧은 쪽 보드는 자기 마지막 프레임 상태를 유지한 채 멈춘다. 범위 밖 seek로 오류가 나면 안 된다.
- 재생 컨트롤(`PlaybackControls`)·노트 마커·사이드바는 **한 벌만** 존재하며 현행 동작 유지(마커 클릭 → 노트 뷰어는 보드와 무관).

## 4. 목표 동작 — 스왑과 시뮬레이터 진입

- 두 보드 사이에 **위치 스왑 버튼**을 둔다. 스왑은 **화면 배치만** 바꾼다 — 각 보드가 나타내는 플레이어 인덱스, 노트의 `origin.player`, 마커·사이드바 해석은 영향받지 않는다(AW-40).
- **시뮬레이터 진입(분기)은 왼쪽 보드만 허용**(#15 확정안). `BranchBar`는 한 벌이고, 분기 시 `player`는 **현재 왼쪽에 놓인 보드의 실제 플레이어 인덱스**다. 원하는 플레이어를 편집하려면 스왑으로 왼쪽에 두고 진입한다.
- 편집 모드 전환은 M6-A의 인플레이스 규칙을 따른다 — 진입 시 재생 크롬(양보드 포함)이 편집 영역으로 교체되고, 종료 시 분기 프레임 복귀.

## 수용 기준 (작업 세션 완료 조건)

- **AW-37 양보드 재생**: 1vs1(ttrm)에서 두 플레이어의 보드+공통 HUD가 동시 재생된다. 솔로(ttr)는 단일 보드 유지.
- **AW-38 동기 컨트롤**: 재생·일시정지·seek·배속·프레임 스텝이 두 보드에 동일 프레임으로 적용된다 — 정지 상태에서 두 컨트롤러 frame 동일, 총프레임 상이 시 max 기준 슬라이더 + 짧은 쪽 마지막 상태 유지.
- **AW-39 보드 스왑**: 두 보드 위치 스왑 버튼이 동작하고, 시뮬레이터 진입은 왼쪽 보드(스왑 반영)만 허용된다.
- **AW-40 노트 호환**: `origin.player`가 실제 플레이어 인덱스 기준으로 유지된다 — 스왑 상태와 무관하게 기존 노트·마커가 깨지지 않고, 새 노트의 `origin.player`도 실제 인덱스다.

※ 소유자가 승인한 것만. 여기 없는 기능은 만들지 않는다.
※ 순수 로직(동기 판정·스왑 매핑·max 프레임)은 유닛으로, 양보드 렌더·스왑·진입은 `e2e/m6b.spec.ts`로 — 테스트 이름에 ID 명시(`tools/check-acceptance.mjs` 대조).
