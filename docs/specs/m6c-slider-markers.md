# Tetorial 명세: 재생 슬라이더·노트 마커 시각 통합 (M6-C, @tetorial/web)

> 상태: 작업 중 · 관련 모듈: apps/web (단독)
> 역할: **재생 슬라이더와 노트 마커를 한 디자인으로 통합한다 — 핸들·마커 형태 교체 + 클러스터 호버 드롭다운.**
> 근거: #56 (소유자 방향 제시 2026-07-18: "슬라이더 디자인과 마커가 따로 노는 느낌").

## 1. 경계 / 책임

- 수정 범위는 **apps/web뿐**이며, 그 안에서도 `components/replay/PlaybackControls.tsx`·`lib/markers.ts`(필요 시)·관련 스타일에 한정한다.
- 마커의 **데이터 계산**(collectMarkers*·clusterMarkers·markerRatio)의 의미론은 변경하지 않는다 — 표시·상호작용만 바꾼다.
- 스키마·딥링크 무관. 노트 열람 흐름(`onMarkerClick` → 뷰어)의 목적지는 현행 유지.

## 2. 현행 구조 (출발점)

- `PlaybackControls`의 `.scrubber`는 기본 `<input type="range">`(원형 네이티브 핸들), 마커는 원형 버튼(`.marker`, `border-radius: 50%`)으로 트랙 위에 절대 배치. 클러스터(2개 이상)는 숫자 표시 + `title` 툴팁뿐이고, **클릭은 항상 첫 노트만 연다**(c.markers[0]) — 다른 노트는 선택 불가.

## 3. 목표 동작

- **핸들(AW-42)**: 슬라이더 핸들을 원이 아니라 **세로 직사각형**으로 (`::-webkit-slider-thumb`·`::-moz-range-thumb` 커스텀).
- **마커(AW-43)**: 원형 대신 **위로 뾰족한 화살촉(삼각형 계열)** 모양으로 트랙을 가리킨다. 슬라이더와 같은 시각 언어(토큰 색·크기 체계)로 통합. 클러스터의 노트 수는 계속 식별 가능해야 한다.
  - 형태 세부는 게이트 11항(소유자 실조작)에서 판정한다 — "화살촉 계열 + 디자인 통합"까지가 계약이고, 세부 형태는 게이트에서 교체될 수 있다.
- **클러스터 드롭다운(AW-44)**: 같은 위치 마커에 노트가 2개 이상이면 **호버(및 키보드 포커스) 시 드롭다운**으로 노트 목록(첫 주석 또는 noteId)을 보여주고, 항목 클릭으로 해당 노트를 연다. 단일 노트 마커는 현행대로 클릭 즉시 연다. `title` 툴팁 의존은 제거해도 된다.
- 기존 `data-testid`(`note-marker`·`scrubber`) 유지. 드롭다운 항목에는 새 testid를 부여한다.

## 수용 기준 (작업 세션 완료 조건)

- **AW-42 슬라이더 핸들**: 재생 슬라이더 핸들이 원형이 아니라 세로 직사각형이다.
- **AW-43 마커 화살촉**: 노트 마커가 위로 뾰족한 화살촉 형태로 트랙을 가리키며 슬라이더와 한 디자인으로 통합된다 — 클러스터 수 식별 유지.
- **AW-44 클러스터 드롭다운**: 노트 2개 이상 마커는 호버/포커스 시 드롭다운으로 노트를 선택해 열 수 있고, 단일 마커는 클릭 즉시 열린다.

※ 소유자가 승인한 것만. 여기 없는 기능은 만들지 않는다.
※ 상호작용(AW-44)은 유닛+`e2e/m6c-markers.spec.ts`로, 형태(AW-42·43)는 e2e에서 관측 가능한 범위(클래스·스타일 속성)로 — 테스트 이름에 ID 명시(`tools/check-acceptance.mjs` 대조).
